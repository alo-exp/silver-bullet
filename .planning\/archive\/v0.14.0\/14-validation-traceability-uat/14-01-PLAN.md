Banner:
```
SILVER BULLET > VALIDATE
```

Step sequence (from 14-RESEARCH.md component design):

**Step 0: Pre-flight checks**
- Read silver-bullet.md section 10 preferences
- Check `.planning/SPEC.md` exists — if missing, emit single FINDING [BLOCK] and exit
- Check at least one PLAN.md exists in `.planning/phases/` — if missing, emit INFO finding and note validation is pre-planning

**Step 1: Read SPEC.md** (NON-SKIPPABLE GATE)
- Parse `## Acceptance Criteria` items into a numbered list
- Parse all `[ASSUMPTION: ...]` blocks with their Status fields
- Parse `## Open Questions` items
- Count total AC items, total assumptions, total open questions

**Step 2: Read PLAN.md files**
- Scan `.planning/phases/` for current phase PLAN.md files
- Extract task names, files_modified, requirement IDs from frontmatter
- Build a coverage map: which PLAN.md tasks address which AC items

**Step 3: Gap Analysis** (NON-SKIPPABLE GATE)
- For each AC item: check if a PLAN.md task addresses it. Missing coverage = FINDING [BLOCK]
- For each [ASSUMPTION] with Status: Follow-up-required: emit FINDING [WARN] with reminder
- For each [ASSUMPTION] with Status: Accepted: emit FINDING [INFO] for awareness
- For PLAN.md tasks with no SPEC.md traceability: emit FINDING [WARN] "orphan task"
- For Open Questions not yet resolved: emit FINDING [WARN]

Finding format (VALD-02) — exactly this line format, one per finding:
```
FINDING [BLOCK] VAL-001: {description}
  Spec ref: {section and item}
  Plan ref: {task or "missing"}
  Resolution: {required action}
```

Codes: VAL-NNN sequential. Severity rules:
- BLOCK: missing AC coverage, unresolved BLOCK-tagged assumptions
- WARN: partial coverage, Follow-up-required assumptions, orphan tasks, open questions
- INFO: accepted assumptions (awareness only)

**Step 4: Surface Assumptions** (VALD-05)
- Print ALL [ASSUMPTION] blocks from SPEC.md as a numbered awareness list
- Header: "## Assumptions Awareness (VALD-05)"
- Each assumption on its own line with Status field visible
- This is mandatory even if all assumptions are Accepted — developer must see them

**Step 5: User Decision Gate** (NON-SKIPPABLE GATE)
- If any BLOCK findings: display them prominently, offer:
  A. Return to /silver:spec to resolve gaps
  B. Show me the BLOCK findings again
- User MUST resolve all BLOCK findings before proceeding. Do NOT allow bypass.
- If only WARN/INFO findings: display summary, offer:
  A. Accept and proceed
  B. Return to /silver:spec to resolve

**Step 6: Write .planning/VALIDATION.md**
- Write all findings in the exact FINDING line format
- Include frontmatter: spec-version, validation-date, finding-counts (block, warn, info)
- This file is consumed by pr-traceability.sh (Plan 02) to populate PR description deferred items

**Step 7: Summary banner**
```
VALIDATION COMPLETE
  Findings: {B} BLOCK, {W} WARN, {I} INFO
  Assumptions surfaced: {N}
  Status: {PASS (no BLOCK) | BLOCKED (has BLOCK findings)}
```
