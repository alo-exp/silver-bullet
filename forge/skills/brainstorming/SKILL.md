# Brainstorming — Product Ideation

## Purpose
Generate and evaluate multiple approaches to a problem. Choose the best path before investing in implementation.

## Steps

### Step 1: Define the Problem
State clearly:
- What problem are we solving?
- Who experiences this problem?
- What are the constraints? (time, budget, technology)
- What does success look like?

### Step 2: Generate Approaches
List 2-4 distinct approaches. For each:
- **Name**: Give it a clear name
- **Description**: How does it work?
- **Pros**: What are the benefits?
- **Cons**: What are the drawbacks?
- **Complexity**: Estimate 1-5

### Step 3: Evaluate Trade-offs
For each approach, assess against:
- **Value**: How much does it solve the problem?
- **Effort**: How much work to implement?
- **Risk**: What could go wrong?
- **Future**: How does it affect future options?

### Step 4: Make a Recommendation
Be decisive. Pick the approach that best fits the constraints with clear reasoning.

### Step 5: Document
Write the brainstorm results to a file:
```
# Brainstorm: <Feature Name>

## Problem Statement
<clear description>

## Approaches Considered

### Approach A: <Name>
- Description: <how it works>
- Pros: <benefits>
- Cons: <drawbacks>
- Complexity: <1-5>

### Approach B: <Name>
...

## Recommendation
**Approach <X>**: <reasoning>

## Next Steps
- Proceed to writing-plans skill
```

## When Done
When you have 2+ approaches documented with a recommendation, the brainstorm is complete.

## Exit Condition
Documented approaches with pros/cons and a recommendation.
