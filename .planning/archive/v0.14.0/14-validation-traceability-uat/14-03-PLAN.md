**hooks/uat-gate.sh** — PreToolUse on Skill matcher (UATG-03, UATG-04):

IMPORTANT: Use Skill matcher, NOT Bash (Pitfall 2 from research). gsd-complete-milestone is invoked via Skill tool.

Boilerplate: same as spec-floor-check.sh (set -euo pipefail, trap 'exit 0' ERR, umask 0077, jq check).

For Skill matcher, the tool_input field is different:
```bash
skill=$(printf '%s' "$input" | jq -r '.tool_input.skill // .tool_input.skillName // ""')
```

Logic:
1. Extract skill name from input
2. Check if skill matches gsd-complete-milestone: `if ! printf '%s' "$skill" | grep -qE 'gsd-complete-milestone|gsd:complete-milestone'; then exit 0; fi`
3. Check `.planning/UAT.md` exists:
   - If missing: emit_block "UAT GATE: .planning/UAT.md not found. Generate UAT checklist from SPEC.md acceptance criteria before completing milestone. Run /silver:feature Step 17."
4. Check for FAIL results (UATG-03):
   - `if grep -qE '\| FAIL \|' .planning/UAT.md; then` count and block
   - Block message: "UAT GATE: {N} criterion/criteria marked FAIL in .planning/UAT.md. Resolve all failures before completing milestone."
5. Check for NOT-RUN results (advisory, not blocking):
   - If NOT-RUN found: emit warning message but do NOT block
6. Spec version check (UATG-04):
   - Read spec-version from UAT.md frontmatter: `grep -m1 '^spec-version:' .planning/UAT.md | awk '{print $2}'`
   - Read spec-version from .planning/SPEC.md: `grep -m1 '^spec-version:' .planning/SPEC.md | awk '{print $2}'`
   - If both exist and differ: emit_block "UAT GATE: UAT was run against spec v{uat} but current SPEC.md is v{current}. Re-run UAT against the current spec."
7. If all checks pass: emit advisory "UAT gate passed. Milestone completion allowed."

Make executable: `chmod +x hooks/uat-gate.sh`

```markdown
## Spec Lifecycle

Silver Bullet anchors every implementation to a verified spec. The spec lifecycle flows:

**Create:** `/silver:spec` (Socratic elicitation) or `/silver:ingest` (external artifact ingestion from JIRA/Figma/Google Docs)

**Artifacts:**
- `.planning/SPEC.md` — canonical spec with YAML frontmatter (`spec-version:`, `jira-id:`, `status:`)
- `.planning/DESIGN.md` — structured design definitions (when Figma input provided)
- `.planning/REQUIREMENTS.md` — derived requirement IDs (REQ-XX, NFR-XX)
- `.planning/SPEC.main.md` — read-only cache of remote spec (cross-repo mode only)

**Validate:** `/silver:validate` performs gap analysis between SPEC.md and PLAN.md before implementation. Findings use severity levels:
- **BLOCK** — missing acceptance criteria coverage or unresolved assumptions. Stops workflow.
- **WARN** — partial coverage, deferred items. Surfaced in PR description.
- **INFO** — awareness items (accepted assumptions).

**Trace:** After `gsd-ship` creates a PR, `pr-traceability.sh` auto-appends spec reference, requirement IDs, and deferred items to the PR description. SPEC.md `## Implementations` section is updated with PR URL post-creation.

**UAT Gate:** Before `gsd-complete-milestone`, UAT.md must exist with all criteria PASS. `uat-gate.sh` blocks if UAT is missing, any criterion is FAIL, or UAT was run against a stale spec version.

**MCP Prerequisites (for /silver:ingest):**
- Atlassian MCP — JIRA ticket + Confluence page ingestion (use `/v1/mcp` streamable HTTP endpoint)
- Figma MCP (beta) — design context and token extraction
- Google Drive MCP — document text extraction (community connector or WebFetch fallback)

If a connector is unavailable, ingestion continues with `[ARTIFACT MISSING]` blocks — no hard block on missing connectors.
```
