### Advisory findings (for backlog)

1. **`.silver-bullet.json` drift from template default.** Repo config at `.silver-bullet.json` has 13-skill `required_deploy` and no `required_deploy_devops`; template default ships 15 skills (adds `review-loop-pass-1/2`) + devops variant. Because stop-check treats the project config as sole-source-of-truth when present, this project runs with a weaker gate than consumers. Sync `.silver-bullet.json` to the canonical 15-skill lists.

2. **Hardcoded fallback in `stop-check.sh` lines 199–200 re-introduces CONS-02 drift vector.** The CONS-02 regression test (`test-required-skills-consistency.sh`) only inspects `hooks/lib/required-skills.sh` for a hardcoded literal — it does not catch a hardcoded list in `stop-check.sh`. Extend the test to grep the hooks dir, or move the fallback into `required-skills.sh` so the single assertion covers both.

3. **Git-history webhook not yet rotated/rewritten (partial SEC-01).** `secret-scan.yml` uses `--no-git`, so the live token remains discoverable in history. Tracked as issue #24 awaiting user-approved `git filter-repo` + Google Chat console rotation. Not a ship-blocker for v0.22.0 since the working-tree token was already removed and CI prevents re-introduction.

4. **TOCTOU window in `nofollow-guard.sh`.** `[[ -L ]]` check and subsequent write are not atomic; an attacker with write access to `~/.claude/.silver-bullet/` could race the guard. Risk is low (umask 0077 scopes the dir to the user), but `O_NOFOLLOW`-style open via a small helper would close it. File for post-ship if we ever loosen dir perms.

5. **`test-skill-refs.sh` external-plugin catalog is hand-maintained.** Adding/renaming an upstream skill requires touching the EXTERNAL_SKILLS array. Consider generating from installed plugin manifests at test time as a stretch improvement.
