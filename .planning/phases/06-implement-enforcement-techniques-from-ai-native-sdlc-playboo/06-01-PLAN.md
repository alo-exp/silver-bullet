  <files>hooks/prompt-reminder.sh</files>
  <action>
Create hooks/prompt-reminder.sh. This must be FAST (< 200ms) -- minimal processing.

1. Shebang: `#!/usr/bin/env bash`, `set -euo pipefail`, `umask 0077`
2. jq check: if missing, exit 0 silently (don't slow down prompts with warnings)
3. DO NOT read stdin -- UserPromptSubmit hooks should not block on stdin. The hook receives context but for speed we only need the state file and config.
4. Resolve .silver-bullet.json by walking up from $PWD (same walk-up pattern)
5. No config -> exit 0 silently
6. Read required_deploy list and state_file path from config (single jq call for speed)
7. Security: validate state_file path within ~/.claude/
8. Check trivial file bypass -> exit 0 if trivial
9. Read state file, compute missing skills (simple loop: for each required skill, grep -qx)
10. If no missing skills: emit `{"hookSpecificOutput":{"additionalContext":"Silver Bullet: all required skills complete."}}`
11. If missing skills exist: build compact list and emit `{"hookSpecificOutput":{"additionalContext":"Silver Bullet -- Missing: skill-a, skill-b, skill-c (N of M complete)"}}`
    - Use comma-separated list, not bullets (compact for context injection)
    - Include count: "(N of M complete)" where N = completed count, M = total required
12. Error trap: on any failure, exit 0 silently (never slow down user prompts)
13. chmod +x

Performance: single jq call to read config, single cat of state file, bash loop for comparison. No git operations, no complex parsing.
