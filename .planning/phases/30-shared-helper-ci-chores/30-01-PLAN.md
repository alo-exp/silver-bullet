```yaml
      - name: Check plugin.json version vs latest git tag
        continue-on-error: true
        run: |
          plugin_version=$(jq -r '.version' .claude-plugin/plugin.json)
          latest_tag=$(git describe --tags --abbrev=0 2>/dev/null || echo "")
          if [[ -z "$latest_tag" ]]; then
            echo "No git tags found — skipping version check"
            exit 0
          fi
          # Strip leading 'v' from tag for comparison (tags are v0.20.11, plugin.json is 0.20.11)
          tag_version="${latest_tag#v}"
          if [[ "$plugin_version" != "$tag_version" ]]; then
            echo "::warning::plugin.json version ($plugin_version) does not match latest git tag ($latest_tag)"
          else
            echo "plugin.json version ($plugin_version) matches latest tag ($latest_tag)"
          fi
```

Key details:
- `continue-on-error: true` ensures the build never fails from this step (per CI-02 requirement)
- Uses `git describe --tags --abbrev=0` to find the most recent tag
- Handles the case where no tags exist (fresh clone, PR from fork) by skipping gracefully
- Strips leading `v` from the tag since plugin.json uses bare version numbers (`0.20.11`) while git tags use `v`-prefix (`v0.20.11`)
- Uses GitHub Actions `::warning::` annotation to make the mismatch visible in the Actions UI (yellow warning icon)
