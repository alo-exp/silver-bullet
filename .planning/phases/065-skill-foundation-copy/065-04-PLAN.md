  <action>
    Clone the public Anthropic knowledge-work-plugins repo into a temp directory, copy each SKILL.md with a plugin-namespace prefix, then remove the temp clone.

    Steps:
    1. Create a temp working directory and clone:
       ```bash
       TMP=$(mktemp -d)
       trap 'rm -rf "$TMP"' EXIT
       git clone --depth 1 https://github.com/anthropics/knowledge-work-plugins.git "$TMP/kwp"
       [ -d "$TMP/kwp" ] || { echo "FAIL: clone failed"; exit 1; }
       ```
    2. Define the canonical lists (per phase context — must match exactly):
       ```bash
       ENGINEERING=(system-design documentation deploy-checklist testing-strategy architecture standup tech-debt code-review debug incident-response)
       DESIGN=(design-critique research-synthesis user-research design-handoff design-system ux-copy accessibility-review)
       PRODUCT_MGMT=(metrics-review synthesize-research write-spec competitive-brief sprint-planning product-brainstorming stakeholder-update roadmap-update)
       MARKETING=(campaign-plan competitive-brief brand-review performance-report seo-audit draft-content content-creation email-sequence)
       ```
    3. Copy each plugin's skills with namespace prefix:
       ```bash
       copy_plugin () {
         local plugin="$1"; shift
         local skills=("$@")
         local missing=0
         for s in "${skills[@]}"; do
           local SRC="$TMP/kwp/$plugin/skills/$s/SKILL.md"
           local DEST="forge/skills/${plugin}-${s}/SKILL.md"
           if [ ! -f "$SRC" ]; then
             echo "WARN: missing $SRC (will fail acceptance check)"
             missing=$((missing+1))
             continue
           fi
           mkdir -p "forge/skills/${plugin}-${s}"
           cp "$SRC" "$DEST"
         done
         echo "$plugin: copied $((${#skills[@]} - missing))/${#skills[@]}"
         return $missing
       }

       copy_plugin engineering "${ENGINEERING[@]}"
       copy_plugin design "${DESIGN[@]}"
       copy_plugin product-management "${PRODUCT_MGMT[@]}"
       copy_plugin marketing "${MARKETING[@]}"
       ```
    4. Verify count and frontmatter sanity:
       ```bash
       NS_COUNT=$( (find forge/skills -maxdepth 1 -type d -name "engineering-*" -o -name "design-*" -o -name "product-management-*" -o -name "marketing-*") | wc -l | tr -d ' ')
       [ "$NS_COUNT" = "33" ] || { echo "FAIL: expected 33 namespaced dirs, found $NS_COUNT"; exit 1; }

       # Spot-check a few key skills
       grep -q "^name: code-review" forge/skills/engineering-code-review/SKILL.md
       grep -q "^name: design-critique" forge/skills/design-design-critique/SKILL.md
       grep -q "^name: write-spec" forge/skills/product-management-write-spec/SKILL.md
       grep -q "^name: campaign-plan" forge/skills/marketing-campaign-plan/SKILL.md
       ```
    5. Cleanup is automatic via `trap` on EXIT.

    **Note on `competitive-brief` collision:** Both `product-management` and `marketing` plugins have a `competitive-brief` skill. The namespace prefix resolves this — they land at `product-management-competitive-brief/` and `marketing-competitive-brief/` respectively. No additional handling needed.

    **Note on frontmatter:** Anthropic's knowledge-work skills already use the standard `name:`/`description:` frontmatter. Per ForgeCode docs the format is identical — no conversion. We do NOT rewrite the inner `name:` field to match the namespaced directory; the YAML `name` reflects the skill's identity, the directory provides the namespace. (If Forge's loader requires directory and frontmatter `name` to match, that is a Phase 68 installer concern and out of scope here.)
