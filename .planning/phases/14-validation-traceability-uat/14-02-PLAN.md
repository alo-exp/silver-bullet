**hooks/spec-session-record.sh** — SessionStart hook (TRAC-01):

Follow the exact boilerplate from spec-floor-check.sh (set -euo pipefail, trap, umask, jq check). This is a SessionStart hook so it does NOT receive tool_input — it fires on session start/clear/compact.

Logic:
1. Check `.planning/SPEC.md` exists — if not, exit 0 silently
2. Extract spec-version: `grep -m1 '^spec-version:' .planning/SPEC.md | awk '{print $2}'`
3. Extract jira-id: `grep -m1 '^jira-id:' .planning/SPEC.md | awk '{print $2}' | tr -d '"'`
4. Write to `${HOME}/.claude/.silver-bullet/spec-session`:
   ```
   spec-version={value}
   jira-id={value}
   ```
5. Exit 0 with advisory: `{"hookSpecificOutput":{"message":"Spec session: SPEC.md v{version}, JIRA: {id}"}}`

If spec-version or jira-id are empty, write empty values — do not block.

**hooks/pr-traceability.sh** — PostToolUse on Bash (TRAC-02, TRAC-03, TRAC-04):

Follow the exact boilerplate from spec-floor-check.sh. This is a PostToolUse hook on Bash matcher.

Logic:
1. Extract command: `cmd=$(printf '%s' "$input" | jq -r '.tool_input.command // ""')`
2. Detect `gh pr create` in the command: `if ! printf '%s' "$cmd" | grep -qE '\bgh pr create\b'; then exit 0; fi`
   - Do NOT trigger on gsd-ship itself — trigger on the actual `gh pr create` Bash command that gsd-ship's sub-agent runs. This avoids Pitfall 1 from research.
3. Read spec-session file: `${HOME}/.claude/.silver-bullet/spec-session`
   - If missing, exit 0 silently (no spec session = no traceability)
4. Extract spec-version and jira-id from spec-session file
5. Read WARN findings from `.planning/VALIDATION.md` if it exists:
   ```bash
   warn_items=""
   if [[ -f ".planning/VALIDATION.md" ]]; then
     warn_items=$(grep 'FINDING \[WARN\]' .planning/VALIDATION.md || true)
   fi
   ```
6. Get PR number from the tool output (PostToolUse has access to tool result):
   ```bash
   pr_url=$(/opt/homebrew/bin/gh pr view --json url --jq '.url' 2>/dev/null || true)
   ```
   If no PR URL found, exit 0 with advisory.
7. Read existing PR body: `/opt/homebrew/bin/gh pr view --json body --jq '.body'`
8. Append traceability block (NEVER overwrite — Pitfall 4):
   ```bash
   traceability_block=$(cat <<TRACE
   ---
   ## Spec Traceability (auto-generated by Silver Bullet)
   - Spec: .planning/SPEC.md (v${spec_version})
   - JIRA: ${jira_id:-n/a}
   - Requirements covered: see SPEC.md ## Acceptance Criteria

   ### Deferred items (WARN findings from silver-validate)
   ${warn_items:-None}
   TRACE
   )
   new_body="${existing_body}

   ${traceability_block}"
   ```
   Write new_body to a temp file, use `/opt/homebrew/bin/gh pr edit --body-file "$tmpfile"`
9. TRAC-04: Update SPEC.md Implementations section:
   ```bash
   today=$(date '+%Y-%m-%d')
   entry="- PR: ${pr_url} | Date: ${today} | Spec-version: ${spec_version}"
   ```
   Use awk or sed to insert the entry after the `<!-- Populated automatically` comment in `## Implementations` section. Then:
   ```bash
   git add .planning/SPEC.md
   git commit -m "trace: link PR to SPEC.md v${spec_version}"
   ```
10. Emit advisory on success: `{"hookSpecificOutput":{"message":"Spec traceability added to PR description."}}`

Both scripts must be executable: `chmod +x hooks/spec-session-record.sh hooks/pr-traceability.sh`
  </action>

  </verify>
