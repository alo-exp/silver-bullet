## Forge Skill Format (MANDATORY)

Every ported skill MUST use exactly this frontmatter — NOT the SB/Claude Code format:

```yaml
---
id: skill-name
title: Human Readable Title
description: One sentence description for Forge skill picker
trigger:
  - "trigger phrase 1"
  - "trigger phrase 2"
---
```

Body: Plain markdown instructions. Instructions are addressed directly to the Forge agent.

## Adaptations Required for ALL Skills

1. **Remove `Skill(skill="...")` calls** — replace with "run the X skill" (trigger phrase invocation)
2. **Remove `AskUserQuestion` tool calls** — replace with "ask the user: [question]" then "wait for response" or structure as a direct inline question
3. **Remove hook references** — any mention of completion-audit, stop-check, record-skill, trivial bypass, ~/.claude/.silver-bullet/, session-init files — delete these entirely
4. **Remove `silver-bullet.md §N` references** — Forge has no such document. Replace with inline instruction text.
5. **Remove plugin-path resolution** (`${PLUGIN_ROOT}/...`) — Forge skills are self-contained; reference forge/skills/ paths directly
6. **Remove Phase -1 Session Init** from silver-init (the ~/.claude/.silver-bullet/session-init check) — Forge has no session state files
7. **Remove plugin install checks** (superpowers, design, engineering, GSD) from silver-init — Forge users install skills via forge-sb-install.sh instead
8. **Remove installed_plugins.json version checks** from silver-init — no plugin registry in Forge
9. **Replace `/compact` Skill invocation** in silver-init Phase -1 with a direct instruction to summarize loaded context
10. **Replace §10 preference loading bash grep** — remove the silver-bullet.md §10 pre-flight read; Forge agents apply best judgment inline
11. **Preserve ALL substantive workflow steps, quality checks, and decision logic**
12. **Preserve all bash commands** that work against the project filesystem (git, find, grep, etc.)
