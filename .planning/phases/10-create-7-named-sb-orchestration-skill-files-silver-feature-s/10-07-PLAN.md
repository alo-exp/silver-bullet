# /silver:fast — Trivial Change Fast Path

SB fast-path for trivial changes only. Bypasses all workflow steps — no planning, no quality gates, no brainstorming, no review cycle.

**This skill is only appropriate for:**
- Typo or text fix
- Config value change (single key/value)
- Variable or symbol rename (≤3 files)
- One-liner code change
- Comment update

**Not appropriate for:**
- Any change touching >3 files
- Logic changes (even "small" ones)
- Dependency additions or removals
- Schema changes
- Changes with downstream impact

If in doubt, use the appropriate named workflow instead.

## Step 1: Complexity Triage Gate

Before invoking gsd-fast, confirm the change is truly trivial:

Ask:

> Confirming trivial scope. Does this change:
>
> A. Touch ≤3 files AND is a typo/config/rename/one-liner? → PROCEED to fast path
> B. Touch >3 files OR involves logic/dependency/schema changes? → ESCALATE to full workflow

If B: do NOT invoke gsd-fast. Instead:
- Identify the appropriate workflow: silver:feature, silver:bugfix, silver:ui, silver:devops, or silver:research
- Display: "This change exceeds the trivial threshold. Routing to [workflow name] instead."
- Invoke the appropriate named workflow via the Skill tool and exit silver:fast.

**If the scope of the change is ambiguous** and cannot be determined without investigation:
- Treat as B (non-trivial) and escalate to the most likely named workflow.
- Do not guess trivial — false trivial classifications cause under-reviewed changes.

## Step 2: Execute Fast Path

**Only if Step 1 confirms A (trivial scope confirmed):**

Invoke `gsd-fast` via the Skill tool. Pass `$ARGUMENTS` as the change description.

## Step 3: STOP Condition — Scope Expansion Check

**During or after gsd-fast execution**, if scope expands beyond 3 files:

STOP. Do not complete the change under the fast path. Display:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 SILVER BULLET ► FAST PATH STOP
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
