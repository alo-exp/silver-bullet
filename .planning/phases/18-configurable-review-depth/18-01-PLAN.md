<task type="auto">
  <name>Task 1: Add review_depth config schema to config.json</name>
  <files>.planning/config.json</files>
  <action>
Add a `review_depth` object to `.planning/config.json` that maps artifact type keys to depth levels. The artifact type keys use the reviewer skill names from the artifact-to-reviewer mapping in SKILL.md (e.g., "review-spec", "review-roadmap", "review-requirements", "review-context", "review-research", "review-design", "review-ingestion-manifest", "review-uat", "gsd-plan-checker", "gsd-code-reviewer", "gsd-verifier", "silver:security").

Schema:
```json
{
  "review_depth": {
    "<reviewer-skill-name>": "deep" | "standard" | "quick"
  }
}
```

Depth semantics:
- `deep`: full QC checks + 2 consecutive clean passes (per ARVW-11b)
- `standard`: full QC checks + 1 clean pass (per ARVW-11c) -- THIS IS THE DEFAULT when no entry exists (per ARVW-11e)
- `quick`: structural checks only + 1 pass (per ARVW-11d)

Add the `review_depth` key to the existing config.json as an empty object `{}` (meaning all artifact types default to `standard`). Add a comment-like documentation key `_review_depth_help` explaining the schema inline:

```json
"_review_depth_help": "Map reviewer skill names to depth: deep (full QC + 2 passes), standard (full QC + 1 pass, default), quick (structural only + 1 pass)",
"review_depth": {}
```

Do NOT pre-populate any artifact type entries -- the empty object means "all standard" which is the correct default per ARVW-11e.
  </action>
  <verify>
    <automated>cat .planning/config.json | python3 -c "import sys,json; d=json.load(sys.stdin); assert 'review_depth' in d; assert isinstance(d['review_depth'], dict); print('PASS')"</automated>
  </verify>
  <done>config.json contains a valid review_depth object (empty = all standard default)</done>
</task>

<task type="auto">
  <name>Task 2: Update review-loop.md with depth-aware algorithm</name>
  <files>skills/artifact-reviewer/rules/review-loop.md</files>
  <action>
Modify the review loop algorithm in Section 1 of review-loop.md to be depth-aware. The changes are:

1. **Add a new subsection before the Algorithm**: "### Depth Resolution" that explains how depth is determined:
   - Read `.planning/config.json` and look up `review_depth[reviewer_skill_name]`
   - If no entry for this reviewer, default to `"standard"` (per ARVW-11e)
   - If no `review_depth` key in config, default to `"standard"` for all
   - If `.planning/config.json` does not exist, default to `"standard"` for all

2. **Modify the Algorithm pseudocode** to incorporate depth:
   - Replace the hardcoded `while consecutive_passes < 2:` with:
     ```
     depth = resolve_depth(reviewer_skill_name)
     required_passes = 2 if depth == "deep" else 1
     check_mode = "structural" if depth == "quick" else "full"
     ```
   - The loop becomes `while consecutive_passes < required_passes:`
   - When invoking the reviewer, pass `check_mode` so reviewers know whether to run full QC or structural-only checks:
     ```
     findings = invoke_reviewer(artifact_path, source_inputs, check_mode)
     ```

3. **Update the "Key Rules" subsection**:
   - Change "A single clean pass is NOT sufficient" to: "The required number of consecutive clean passes depends on depth: deep=2, standard=1, quick=1"
   - Add: "When check_mode is 'structural', reviewers MUST skip content quality checks and only validate section presence and format validity (per ARVW-11d)"
   - Add: "Default depth is 'standard' for all artifact types when review_depth is absent or does not contain an entry for the reviewer (per ARVW-11e)"
   - Keep the safety cap at 5 rounds unchanged

4. **Update the display messages** to include depth context:
   - On loop start: `display "Reviewing {artifact} at {depth} depth (requires {required_passes} consecutive clean pass(es))"`
   - On clean pass: Update to `"Clean pass {consecutive_passes}/{required_passes}..."` instead of hardcoded `/2`
   - On completion: `"Review complete at {depth} depth. {required_passes} consecutive clean pass(es) achieved."`

5. **Update Section 3 (Audit Trail)** format to include depth:
   - Add `- **Depth:** {depth}` and `- **Check mode:** {check_mode}` to the round template
   - Change `**Consecutive clean passes:** {count}/2` to `**Consecutive clean passes:** {count}/{required_passes}`

Do NOT change Section 2 (State Tracking) format -- the state file structure remains the same. The `consecutive_passes` and `round` fields still work identically.

IMPORTANT: The existing `while consecutive_passes < 2` behavior maps to `deep` depth. The NEW default `standard` requires only 1 pass. This is a deliberate change from the Phase 15 behavior -- ARVW-11e specifies standard as the default, and ARVW-11c defines standard as 1 clean pass.
  </action>
  <verify>
    <automated>grep -c "resolve_depth\|required_passes\|check_mode\|structural" skills/artifact-reviewer/rules/review-loop.md | xargs test 4 -le</automated>
  </verify>
  <done>review-loop.md algorithm uses depth resolution: deep=2 passes full QC, standard=1 pass full QC (default), quick=1 pass structural only. Display messages and audit trail reflect depth.</done>
</task>
