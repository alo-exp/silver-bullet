**Create `skills/artifact-reviewer/rules/reviewer-interface.md`:**

Define the standard interface contract that ALL reviewers must implement:

**Input contract:**
```
Reviewer receives:
  artifact_path: string     — absolute or relative path to the artifact file
  source_inputs: string[]   — optional paths to source artifacts (e.g., SPEC.md for DESIGN.md review)
  review_context: string    — optional additional context (e.g., "Phase 16 spec creation")
```

**Output contract:**
```
Reviewer returns structured findings:
  status: "PASS" | "ISSUES_FOUND"
  findings: Finding[]

Finding:
  id: string          — unique finding ID (e.g., "SPEC-F01")
  severity: "ISSUE"   — only ISSUE severity blocks; INFO is advisory
  description: string — what is wrong
  location: string    — section or line reference in the artifact
  suggestion: string  — how to fix
```

**Reviewer responsibilities:**
- Read the artifact at artifact_path
- If source_inputs provided, cross-reference for consistency
- Validate against the artifact type's quality criteria (defined per-reviewer)
- Return structured findings — NEVER return free-form text as the primary output
- A "PASS" result means zero ISSUE findings — INFO findings are allowed with PASS

**Reviewer prohibitions:**
- Do NOT modify the artifact (reviewers are read-only)
- Do NOT skip sections because they "look fine" — validate every required section
- Do NOT return PASS when any required section is missing or empty

**Create `skills/artifact-reviewer/rules/review-loop.md`:**

This file defines the reusable review loop mechanism, per-artifact state tracking, and REVIEW-ROUNDS.md audit trail.

**Section 1: Review Loop Mechanism (ARFR-02)**

The loop algorithm (pseudocode for Claude to follow):

```
consecutive_passes = 0
round = 1

# Resume from state if available (ARFR-03)
state = load_review_state(artifact_path)
if state exists:
  consecutive_passes = state.consecutive_passes
  round = state.round
  display "Resuming review of {artifact} from round {round} ({consecutive_passes} consecutive passes so far)"

while consecutive_passes < 2:
  findings = invoke_reviewer(artifact_path, source_inputs)
  
  record_round(artifact_path, round, findings)  # ARFR-04
  save_review_state(artifact_path, round, consecutive_passes)  # ARFR-03
  
  if findings.status == "PASS":
    consecutive_passes += 1
    if consecutive_passes < 2:
      display "Clean pass {consecutive_passes}/2. Re-reviewing for confirmation..."
      round += 1
  else:
    consecutive_passes = 0  # Reset on any failure
    display "Round {round}: {len(findings.findings)} issue(s) found. Fixing..."
    
    # Fix each finding
    for finding in findings.findings:
      apply_fix(artifact_path, finding)
    
    round += 1

display "2 consecutive clean passes achieved. Review complete."
clear_review_state(artifact_path)
```

Key rules:
- A single clean pass is NOT sufficient — the loop continues
- Any ISSUE finding resets the consecutive pass counter to 0
- INFO findings do NOT reset the counter (INFO is advisory)
- The loop is self-limiting: it terminates after 2 consecutive PASS results
- If the loop reaches 5 rounds without 2 consecutive passes, surface to user with all accumulated findings

**Section 2: Per-Artifact State Tracking (ARFR-03)**

State file location: `~/.claude/.silver-bullet/review-state/{artifact-hash}.json`

Where `{artifact-hash}` = first 8 chars of SHA256 of the artifact's absolute path.

State file format:
```json
{
  "artifact_path": "/absolute/path/to/artifact.md",
  "reviewer": "reviewer-skill-name",
  "round": 3,
  "consecutive_passes": 1,
  "last_updated": "2026-04-09T12:00:00Z",
  "findings_history": [
    { "round": 1, "status": "ISSUES_FOUND", "finding_count": 2 },
    { "round": 2, "status": "PASS", "finding_count": 0 },
    { "round": 3, "status": "in_progress" }
  ]
}
```

State operations:
- `load_review_state(artifact_path)`: Read state file if exists; return null if not
- `save_review_state(artifact_path, round, consecutive_passes)`: Write/update state file
- `clear_review_state(artifact_path)`: Delete state file after successful 2-pass completion

State is written after EVERY round (not just at completion) so sessions can resume.

**Implementation as shell commands:**
```bash
# State directory
SB_REVIEW_STATE="${HOME}/.claude/.silver-bullet/review-state"
mkdir -p "$SB_REVIEW_STATE"

# Hash for state file
artifact_hash=$(printf '%s' "$(realpath "$artifact_path")" | shasum -a 256 | cut -c1-8)
state_file="${SB_REVIEW_STATE}/${artifact_hash}.json"
```

**Section 3: REVIEW-ROUNDS.md Audit Trail (ARFR-04)**

After each round completes, append to `REVIEW-ROUNDS.md` in the same directory as the reviewed artifact.

If the artifact is `.planning/SPEC.md`, the audit trail is `.planning/REVIEW-ROUNDS.md`.

Format:
```markdown
# Review Rounds

## {artifact filename}

### Round {N} — {ISO timestamp}
- **Reviewer:** {reviewer-skill-name}
- **Status:** {PASS | ISSUES_FOUND}
- **Findings:**
  - {finding.id}: {finding.description} [{finding.severity}]
  - ...
  (or "No issues found" if PASS)
- **Consecutive clean passes:** {count}/2

---
```

Rules:
- REVIEW-ROUNDS.md is append-only during a review session (never truncate prior rounds)
- Each artifact gets its own section (identified by `## {artifact filename}` header)
- If REVIEW-ROUNDS.md already has rounds for this artifact from a prior session, append new rounds after the last one
- Commit REVIEW-ROUNDS.md alongside the artifact after the review loop completes (2 clean passes achieved)
