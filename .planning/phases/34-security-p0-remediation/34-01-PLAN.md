# Phase 34-01 — Security P0 Remediation (SEC-01)

Milestone: v0.22.0
Issue: alo-exp/silver-bullet#24
Requirement: SEC-01 (`.planning/REQUIREMENTS.md`)

## Problem

`.silver-bullet.json` line 53 on origin/main contained a live Google Chat
webhook URL with embedded API key + token. Anyone with the public repo URL
could POST into the Ālo labs chat space. Committed in `ac3e8f7`.

## Tasks

### Autonomous remediation (this phase)

- [x] **Remove webhook from tracked config.** Deleted the
  `notifications.google_chat_webhook` field from `.silver-bullet.json`.
  Replaced with `_notifications_comment` pointing readers to the env-var
  mechanism.
- [x] **Move webhook read to env var.** Updated
  `skills/silver-create-release/SKILL.md` Step 6 to read
  `${SB_GCHAT_WEBHOOK:-}` instead of `jq .notifications.google_chat_webhook`.
  Added a Security note prohibiting commits of webhook URLs.
- [x] **Switch curl payload to `jq`-built JSON.** Raw string interpolation of
  `$version`/`$summary`/`$release_url` into the JSON body was replaced with
  `jq -n --arg … | curl --data-binary @-`. Prevents JSON injection from
  crafted commit messages (defense in depth on top of the existing Step 3
  backtick sanitization).
- [x] **Harden `.gitignore`** with a `Secrets` section covering `.env`,
  `.env.*`, `*.key`, `*.pem`, `id_rsa*`, `secrets.json`.
- [x] **Add gitleaks CI.** New `.github/workflows/secret-scan.yml` runs
  `gitleaks detect --no-git` on every PR and push to main. `--no-git`
  scopes the scan to the working tree so the pre-remediation commit in
  history does not fail every PR until filter-repo runs.
- [x] **Write this PLAN.**

## BLOCKERS (user action required)

### 1. Rotate the leaked webhook

The webhook token committed in `ac3e8f7` is still valid until rotated.
Until the user logs into the Google Chat space settings and regenerates
the incoming-webhook URL, the old token remains a live abuse vector even
after git history is rewritten (anyone who pulled or forked the repo
before remediation retains the token).

**Action:** log into Google Chat → the `AAQA4GJAMVA` space → Apps &
integrations → regenerate the "silver-bullet release" webhook. Export
the new URL as `SB_GCHAT_WEBHOOK` in `~/.zshrc`.

### 2. Rewrite git history to purge the token

The token is still reachable via `git log` / GitHub's commit view. Scrub
with `git-filter-repo` (installed via `brew install git-filter-repo`):

```
# from a fresh clone — filter-repo refuses to run on a repo with a remote
git clone --mirror https://github.com/alo-exp/silver-bullet.git sb-mirror.git
cd sb-mirror.git
git filter-repo --replace-text <(printf '%s\n' \
  'AIzaSyDdI0hCZtE6vySjMm-WEfRq3CPzqKqqsHI==>REDACTED' \
  'N9gDn2-uSiT3xbN-iX7WjFPru5BNr7IBQA3BuKoq7ss==>REDACTED')
git push --force --mirror origin
```

This is destructive — it rewrites every commit SHA after `ac3e8f7`, breaks
in-flight PRs and forks, and requires every collaborator to re-clone. Per
global CLAUDE.md: "NEVER run destructive git commands (push --force,
reset --hard, …) unless the user explicitly requests these actions."

**Action:** user must explicitly request the rewrite, then I will run the
commands in a clean mirror clone.
