<!-- Lock file schema (LOCK-01) — written and read by this helper.
     Plan 03 tests assert against this exact shape. -->

Top-level: JSON object keyed by zero-padded phase number ("070", "071", ...).
Empty file: `{}` (created on first claim).

Per-phase value shape:
{
  "owner_id": "claude-shafqat-mac-23456",   // <runtime>-<hostname>-<pid>
  "agent_runtime": "claude",                 // one of identity_tags
  "claimed_at": "2026-04-28T12:34:56Z",      // ISO 8601 UTC
  "last_heartbeat_at": "2026-04-28T12:35:10Z",
  "host": "shafqat-mac",
  "pid": 23456,
  "intent": "Implement LOCK-02 helper"       // free-form caller-supplied
}

peek output contract:
- Free phase: empty stdout, exit 0
- Held + fresh: valid JSON object on stdout (the per-phase value shape), exit 0
- Held + stale: same JSON with extra top-level "expired": true field, exit 0

CLI surface (the contract Phase 71+ calls):
  phase-lock.sh claim     <phase> <runtime> <intent>
  phase-lock.sh heartbeat <phase> <runtime>
  phase-lock.sh release   <phase> <runtime>
  phase-lock.sh peek      <phase>

Exit codes:
  claim:     0 = acquired (fresh or stolen-stale); 2 = conflict (active lock by another); 3 = unknown runtime; 4 = config/precondition failure
  heartbeat: 0 = updated; 2 = no lock or non-self lock
  release:   0 = released or no-op (no existing lock); 2 = lock held by another runtime
  peek:      0 = always (output indicates state)
  any op:    1 = unexpected error (jq missing, .planning/ missing, malformed JSON)

Env vars honored:
  SB_PHASE_LOCK_FILE       — overrides .planning/.phase-locks.json path (testability)
  SB_PHASE_LOCK_INHERITED  — when "true", claim/heartbeat/release are no-ops exit 0 with INFO stderr; peek unaffected
  SB_DEFAULT_CONFIG        — overrides templates/silver-bullet.config.json.default lookup (testability; matches required-skills.sh)
</interfaces>

4. **Config resolution helper** (mirror `hooks/lib/required-skills.sh` __sb_rs_find_default_config pattern):
   - Locate `templates/silver-bullet.config.json.default`:
     - If `SB_DEFAULT_CONFIG` set and file exists, use it.
     - Else walk up from script dir; the script is at `.planning/scripts/phase-lock.sh`, so `${REPO_ROOT}/templates/silver-bullet.config.json.default` is the canonical location.
   - Project-local override: `${REPO_ROOT}/.silver-bullet.json` if present.
   - `_pl_config_get_tags()`: returns space-separated tags, project-local `.silver-bullet.json` `.multi_agent.identity_tags[]` first, fallback to default template `.multi_agent.identity_tags[]`. Hard fallback if neither exists: `claude forge codex opencode`.
   - `_pl_config_get_ttl()`: returns integer seconds, project-local first, then default template, hard fallback `1800`.

5. **Helper functions:**
   - `_pl_now_iso()`: prints current UTC ISO 8601, e.g. `date -u +%Y-%m-%dT%H:%M:%SZ`
   - `_pl_now_epoch()`: prints current Unix epoch seconds, `date -u +%s`
   - `_pl_iso_to_epoch <iso>`: converts ISO 8601 UTC string back to epoch. Use `date -j -u -f "%Y-%m-%dT%H:%M:%SZ" "$1" +%s` on macOS; fallback `date -u -d "$1" +%s` on Linux. Try GNU first, fall back to BSD via `||`.
   - `_pl_normalize_phase <input>`: zero-pads to 3 digits if numeric (`70` → `070`, `070` → `070`, `2.1` → `2.1` left as-is for milestone-prefixed phases — use `printf '%03d' "$n" 2>/dev/null || printf '%s' "$n"`).
   - `_pl_owner_id <runtime>`: prints `<runtime>-<hostname-short>-<pid>` where hostname-short is `hostname -s 2>/dev/null || hostname`.
   - `_pl_validate_runtime <runtime>`: returns 0 if `$runtime` is in the resolved identity_tags list, else prints `ERR: unknown runtime tag '$runtime' — allowed: <list>` to stderr and returns 3.

6. **Inheritance no-op short-circuit:**
   At the top of `cmd_claim`, `cmd_heartbeat`, `cmd_release` (NOT `cmd_peek`):
   ```bash
   if [[ "${SB_PHASE_LOCK_INHERITED:-}" == "true" ]]; then
     echo "INFO: phase-lock $OP skipped — SB_PHASE_LOCK_INHERITED=true" >&2
     exit 0
   fi
   ```

7. **flock wrapper for mutation:**
   Define `_pl_with_flock()` that opens fd 9 on the sidecar and runs the body under exclusive flock:
   ```bash
   _pl_with_flock() {
     # Ensure sidecar exists for flock(1) to operate on
     : > "$LOCK_SIDECAR" 2>/dev/null || true
     [[ -f "$LOCK_SIDECAR" ]] || { echo "ERR: cannot create flock sidecar at $LOCK_SIDECAR" >&2; return 1; }
     exec 9>"$LOCK_SIDECAR"
     flock -x 9
     "$@"
     local rc=$?
     exec 9>&-
     return $rc
   }
   ```
   If `flock` is missing (rare but possible on stripped systems), exit 1 with `ERR: flock(1) is required`. Check at startup with `command -v flock`.

8. **Atomic JSON read-modify-write:**
   `_pl_read_json()`: if `$LOCK_FILE` does not exist, prints `{}`; else `cat "$LOCK_FILE"`. Validates with `jq empty` — on parse failure, prints `ERR: corrupt lock file at $LOCK_FILE` to stderr and exits 1.
   `_pl_write_json <json-string>`: writes via temp + atomic rename:
   ```bash
   local tmp="${LOCK_FILE}.tmp.$$"
   printf '%s\n' "$1" > "$tmp"
   mv "$tmp" "$LOCK_FILE"
   ```

9. **Operation: `cmd_claim <phase> <runtime> <intent>`**
   - Validate args: phase, runtime, intent all required. Exit 4 with usage on missing.
   - `_pl_validate_runtime "$runtime"` (exits 3 on unknown).
   - Inheritance short-circuit (see step 6).
   - Under flock:
     - `current=$(_pl_read_json | jq --arg p "$phase" '.[$p] // empty')`
     - If empty (no existing lock): write new entry, exit 0.
     - Else: compute staleness:
       - `last_hb=$(printf '%s' "$current" | jq -r '.last_heartbeat_at')`
       - `now_e=$(_pl_now_epoch)`; `hb_e=$(_pl_iso_to_epoch "$last_hb")`
       - `ttl=$(_pl_config_get_ttl)`
       - `age=$((now_e - hb_e))`
       - If `age > ttl`: emit `WARN: stealing stale lock from <prior-owner-id> (heartbeat ${age}s ago, ttl ${ttl}s)` to stderr; overwrite entry; exit 0.
       - Else (active lock): emit `ERR: phase $phase is locked by <prior-owner-id> (runtime=<prior-runtime>, intent=<prior-intent>)` to stderr; exit 2. Do NOT mutate the file.
     - Entry shape (jq construction):
       ```bash
       new_entry=$(jq -n \
         --arg owner_id "$(_pl_owner_id "$runtime")" \
         --arg runtime "$runtime" \
         --arg ts "$(_pl_now_iso)" \
         --arg host "$(hostname -s 2>/dev/null || hostname)" \
         --argjson pid "$$" \
         --arg intent "$intent" \
         '{owner_id:$owner_id, agent_runtime:$runtime, claimed_at:$ts, last_heartbeat_at:$ts, host:$host, pid:$pid, intent:$intent}')
       updated=$(_pl_read_json | jq --arg p "$phase" --argjson v "$new_entry" '.[$p] = $v')
       _pl_write_json "$updated"
       ```

10. **Operation: `cmd_heartbeat <phase> <runtime>`**
    - Validate args. Inheritance short-circuit.
    - Under flock: read current entry. If empty or `agent_runtime` ≠ `$runtime` or `owner_id` ≠ this process's `_pl_owner_id "$runtime"`: stderr `ERR: cannot heartbeat — phase $phase not owned by current runtime/process`; exit 2.
    - Else: update `last_heartbeat_at` to `_pl_now_iso`; write back; exit 0.

11. **Operation: `cmd_release <phase> <runtime>`**
    - Validate args. Inheritance short-circuit.
    - Under flock: read current.
      - If no entry: exit 0 (no-op).
      - If `owner_id` ≠ self: stderr `ERR: cannot release — phase $phase owned by <other-owner-id>`; exit 2; do NOT delete.
      - Else: `updated=$(_pl_read_json | jq --arg p "$phase" 'del(.[$p])')`; write back; exit 0.

12. **Operation: `cmd_peek <phase>`**
    - Validate args (1 arg). NO inheritance short-circuit (peek always works).
    - NO flock required for read (atomic-rename writes mean any single read sees a complete file).
    - Read JSON; extract entry for `$phase`.
    - If empty: print nothing; exit 0.
    - Else compute staleness (as in claim). If stale: emit JSON with `+ {expired: true}` merged in. If fresh: emit JSON as-is.
    - Use jq to ensure compact valid JSON output.

13. **Dispatch:**
    ```bash
    OP="${1:-}"
    shift || true
    case "$OP" in
      claim)     [[ $# -eq 3 ]] || { echo "Usage: phase-lock.sh claim <phase> <runtime> <intent>" >&2; exit 4; }
                 phase=$(_pl_normalize_phase "$1"); _pl_with_flock cmd_claim "$phase" "$2" "$3" ;;
      heartbeat) [[ $# -eq 2 ]] || { echo "Usage: phase-lock.sh heartbeat <phase> <runtime>" >&2; exit 4; }
                 phase=$(_pl_normalize_phase "$1"); _pl_with_flock cmd_heartbeat "$phase" "$2" ;;
      release)   [[ $# -eq 2 ]] || { echo "Usage: phase-lock.sh release <phase> <runtime>" >&2; exit 4; }
                 phase=$(_pl_normalize_phase "$1"); _pl_with_flock cmd_release "$phase" "$2" ;;
      peek)      [[ $# -eq 1 ]] || { echo "Usage: phase-lock.sh peek <phase>" >&2; exit 4; }
                 phase=$(_pl_normalize_phase "$1"); cmd_peek "$phase" ;;
      ""|-h|--help) cat <<EOF
    Usage: phase-lock.sh <op> <phase> [args...]
      claim     <phase> <runtime> <intent>   Acquire phase lock (steals stale)
      heartbeat <phase> <runtime>            Refresh heartbeat (must own)
      release   <phase> <runtime>            Release phase lock (must own)
      peek      <phase>                      Print lock JSON or empty
    Env:
      SB_PHASE_LOCK_FILE        Override lock file path (default: .planning/.phase-locks.json)
      SB_PHASE_LOCK_INHERITED   When "true", claim/heartbeat/release are no-ops
      SB_DEFAULT_CONFIG         Override config template lookup
    EOF
        exit 0 ;;
      *) echo "ERR: unknown op '$OP'" >&2; exit 4 ;;
    esac
    ```
