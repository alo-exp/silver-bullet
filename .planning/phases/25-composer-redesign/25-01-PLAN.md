Insert AFTER Per-Phase Loop, BEFORE Step-Skip Protocol. This section defines the logic that runs BETWEEN path sections:

After each PATH completes:
1. **Exit check** (D-07.1): Verify the PATH's exit condition was met (from path contract in docs/composable-paths-contracts.md). If not met, offer: retry, skip with reason, or insert PATH 14 (DEBUG).
2. **Composition evaluation** (D-07.2): Re-evaluate context for dynamic insertion triggers:
   - If execution failed → insert PATH 14 (DEBUG) before next path (per D-11)
   - If UI files discovered in SUMMARY.md → insert PATH 6 (DESIGN CONTRACT) if not in composition (per D-11, D-12)
   - Record any insertions in WORKFLOW.md Dynamic Insertions table with reason and timestamp (per D-13)

   - **Tier 2 — Permission-stall**: If blocked waiting for user input >5 min in autonomous mode, auto-select recommended option and log to WORKFLOW.md Autonomous Decisions table (per D-17)
   - **Tier 3 — Context exhaustion**: If context window >80%, display `/compact` recommendation. If >90%, state: `Context exhaustion imminent. Running /compact before continuing.` (per D-18)
   - **Tier 4 — Heartbeat sentinel**: Each path invocation writes a heartbeat timestamp to WORKFLOW.md. If heartbeat gap >15 minutes, display: `⚠ HEARTBEAT GAP: PATH {N} may have stalled. Options: [retry/skip/debug]` (per D-19)
4. **Advance** (D-07.4): Move to next path in composition chain
5. **Progress report** (D-09): Display `PATH {current}/{total}: {name} ✓ | Context: ~{percent}% | Remaining: {list}`
6. **WORKFLOW.md update** (D-10): Write path status and timestamp to Path Log table

Anti-stall timers use wall-clock time (Claude's discretion). Heartbeat timestamps use ISO 8601 format.
