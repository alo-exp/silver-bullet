  <name>Task 4 (CHR-04): Strip trailing newlines from RELEASE_NOTES_BODY before printf in silver-create-release</name>
  <read_first>skills/silver-create-release/SKILL.md</read_first>
  <files>skills/silver-create-release/SKILL.md</files>
  <action>
In Step 5 of skills/silver-create-release/SKILL.md, the CHANGELOG head/printf/tail block reads:

```bash
VERSION_BARE="${VERSION#v}"   # strip leading 'v' if present
TODAY=$(date '+%Y-%m-%d')
TMP=$(mktemp)
{
  head -1 CHANGELOG.md
  printf '\n## [%s] — %s\n\n%s\n\n---\n' "$VERSION_BARE" "$TODAY" "$RELEASE_NOTES_BODY"
  tail -n +2 CHANGELOG.md
} > "$TMP" && mv "$TMP" CHANGELOG.md
```

Add a trailing-newline strip for `RELEASE_NOTES_BODY` BEFORE this block. Insert the following line immediately before `VERSION_BARE=`:

```bash
RELEASE_NOTES_BODY=$(printf '%s' "$RELEASE_NOTES_BODY" | sed 's/[[:space:]]*$//')
```

The resulting block should be:

```bash
RELEASE_NOTES_BODY=$(printf '%s' "$RELEASE_NOTES_BODY" | sed 's/[[:space:]]*$//')
VERSION_BARE="${VERSION#v}"   # strip leading 'v' if present
TODAY=$(date '+%Y-%m-%d')
TMP=$(mktemp)
{
  head -1 CHANGELOG.md
  printf '\n## [%s] — %s\n\n%s\n\n---\n' "$VERSION_BARE" "$TODAY" "$RELEASE_NOTES_BODY"
  tail -n +2 CHANGELOG.md
} > "$TMP" && mv "$TMP" CHANGELOG.md
```

This strips trailing whitespace (including newlines) from `RELEASE_NOTES_BODY` before it is passed to `printf`. Without this, if the release notes body ends with a trailing newline (which `git log` output often does), the `\n\n---\n` separator gets an extra blank line prepended, causing formatting issues in CHANGELOG.md.

The `sed 's/[[:space:]]*$//'` expression uses POSIX character classes so it is portable across macOS and Linux.
