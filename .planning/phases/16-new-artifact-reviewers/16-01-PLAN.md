**skills/review-spec/SKILL.md** (ARVW-01):
- Frontmatter: name: review-spec, description of SPEC.md reviewer, argument-hint: "&lt;spec-path&gt; [--source-inputs &lt;jira-ticket&gt; &lt;figma-url&gt;]"
- Loading Rules section: MUST load @skills/artifact-reviewer/rules/reviewer-interface.md and @skills/artifact-reviewer/rules/review-loop.md
- Quality Criteria section with these checks (each producing an ISSUE finding if violated):
  1. Required sections present: Overview, User Stories, UX Flows, Acceptance Criteria, Assumptions, Open Questions, Out of Scope, Implementations — check against templates/specs/SPEC.md.template
  2. Overview is non-empty and contains a problem statement (not just a placeholder like "[2-3 sentence...]")
  3. At least 1 User Story in "As a..., I want to..., so that..." format
  4. At least 1 Acceptance Criterion that is testable (contains measurable/observable language, not vague)
  5. Assumption density: every ASSUMPTION block has Status field (Resolved or Follow-up-required)
  6. Frontmatter completeness: spec-version, status, created, last-updated fields present

- Output Contract section: reminder that output MUST follow the Finding structure from reviewer-interface.md (id prefixed "SPEC-F", severity ISSUE or INFO, description, location, suggestion)
- Reviewer Prohibitions: read-only, no modifications, no skipping sections
