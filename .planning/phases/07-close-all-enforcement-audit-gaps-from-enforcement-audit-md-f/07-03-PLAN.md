**F-15: compliance-status.sh config cache mtime invalidation**

In compliance-status.sh, the cache check block (around lines 32-39) currently checks if cached_path is a valid file. Extend it to also store and check mtime:

Change the cache write (line 59) from:
```bash
printf '%s' "$config_file" > "$cache_file"
```
to:
```bash
config_mtime=$(stat -f '%m' "$config_file" 2>/dev/null || stat -c '%Y' "$config_file" 2>/dev/null || echo "0")
printf '%s\n%s' "$config_file" "$config_mtime" > "$cache_file"
```

Change the cache read (lines 33-38) to read both path and mtime:
```bash
if [[ -f "$cache_file" ]]; then
  cached_path=$(sed -n '1p' "$cache_file")
  cached_mtime=$(sed -n '2p' "$cache_file")
  if [[ -n "$cached_path" && -f "$cached_path" ]]; then
    current_mtime=$(stat -f '%m' "$cached_path" 2>/dev/null || stat -c '%Y' "$cached_path" 2>/dev/null || echo "0")
    if [[ "$cached_mtime" == "$current_mtime" ]]; then
      config_file="$cached_path"
    else
      rm -f "$cache_file"
    fi
  else
    rm -f "$cache_file"
  fi
fi
```

**F-18: Update .silver-bullet.json src_pattern**

Change `"src_pattern": "/src/"` to `"src_pattern": "/hooks/|/skills/|/templates/"` in `.silver-bullet.json`.

NOTE: The src_pattern validation in dev-cycle-check.sh (line 160-162) currently only allows simple path segments: `'^/[a-zA-Z0-9/_.-]*/?$'`. The pipe-separated pattern `/hooks/|/skills/|/templates/` will FAIL this validation and fall back to `/src/`. This is a known issue — the validation regex in dev-cycle-check.sh must be relaxed to allow `|` (pipe) characters for alternation patterns. Update the validation on line 161 from:
```bash
if ! printf '%s' "$src_pattern" | grep -qE '^/[a-zA-Z0-9/_.-]*/?$'; then
```
to:
```bash
if ! printf '%s' "$src_pattern" | grep -qE '^/[a-zA-Z0-9/_.|()-]*/?$'; then
```
This allows pipe `|` for alternation while still blocking dangerous regex characters.
