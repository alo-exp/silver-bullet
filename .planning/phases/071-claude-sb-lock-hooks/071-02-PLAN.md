<!-- Phase 70 helper contract (from .planning/scripts/phase-lock.sh + 070-SUMMARY.md) -->

CLI signatures and exit codes used by these hooks:
```
phase-lock.sh claim     <phase> <runtime> <intent>
   exit 0  acquired (or stolen-stale)
   exit 2  conflict — stderr identifies current owner
   exit 3  unknown runtime tag
   exit 4  usage error
   exit 1  internal error

phase-lock.sh heartbeat <phase> <runtime>
   exit 0  updated
   exit 2  not owned by this runtime+host

phase-lock.sh release   <phase> <runtime>
   exit 0  released or no-op
   exit 2  owned by another runtime/host

phase-lock.sh peek      <phase>
   exit 0  always — empty stdout = free, JSON = held
```

`SB_PHASE_LOCK_INHERITED=true` already short-circuits claim/heartbeat/release inside
phase-lock.sh itself (returns 0 with INFO stderr). The hooks ALSO add their own
top-level short-circuit so manifest writes / throttle touches are skipped under
delegation. Both layers are belt-and-suspenders.

PreToolUse conflict block protocol (per CONTEXT.md §New hook scripts §1):
On lock-conflict, the claim hook emits a stderr block-message and `exit 2`.
Claude Code interprets exit-2 from a PreToolUse hook as a hard block; the
stderr text is surfaced to the user. Do NOT emit `permissionDecision:"deny"`
JSON for the conflict path — that pattern is reserved elsewhere; CONTEXT.md
locks the conflict-path as exit-2 + stderr.

```bash
emit_conflict_block() {
  # $1 = phase, $2 = owner_runtime, $3 = owner_id, $4 = intent, $5 = helper_stderr
  printf '🚫 PHASE LOCK CONFLICT — phase %s is locked by %s (owner_id=%s).\n' "$1" "$2" "$3" >&2
  printf 'Attempted intent: %s\n' "$4" >&2
  [[ -n "$5" ]] && printf '%s\n' "$5" >&2
  printf 'Hint: run `.planning/scripts/phase-lock.sh peek %s` for details.\n' "$1" >&2
  exit 2
}
```

3. **`SB_PHASE_LOCK_INHERITED` short-circuit** (per D-03): if env var equals `"true"`, exit 0 immediately with no side effects. Place this BEFORE any jq probe / stdin read so delegated subagents skip everything.

15. **Call helper**: capture both stdout and stderr.
    ```bash
    set +e
    helper_stderr=$("$helper" claim "$phase" "claude" "$intent" 2>&1 >/dev/null)
    helper_rc=$?
    set -e
    ```
16. **Handle exit codes**:

      ```bash
      owner_id=$(printf '%s' "$owner_json" | jq -r '.owner_id // "?"')

      printf 'Attempted intent: %s\n' "$intent" >&2
      [[ -n "$helper_stderr" ]] && printf '%s\n' "$helper_stderr" >&2
      printf 'Hint: run `.planning/scripts/phase-lock.sh peek %s` for details.\n' "$phase" >&2
      ```

      The message MUST include: phase number, current owner runtime, current owner_id, attempted intent, and the peek hint (per CONTEXT.md §specifics).

Source `hooks/lib/nofollow-guard.sh` and call `sb_safe_write "$manifest"` before the append (this rm-s the file if it's a symlink, then the append creates a real file). Reference: `hooks/lib/nofollow-guard.sh` lines 19-24.

### A. `hooks/phase-lock-heartbeat.sh` (HOOK-02)

After the shared prologue:
1. Read stdin JSON, extract `raw_session_id` and apply the sanitization rule above. Compute `manifest` path the same way as the claim hook.
2. If `manifest` does not exist or is empty, exit 0 (no claimed phases this session — nothing to heartbeat).
3. Read manifest line-by-line. For each `phase` (skip blank lines):
   a. Compute throttle file: `throttle="${HOME}/.claude/.silver-bullet/heartbeat-${phase}"`. Validate under `${HOME}/.claude/` (same path-validation pattern as the manifest).
   b. **Throttle check** (per CONTEXT.md §Heartbeat throttle):
      ```bash
      now=$(date +%s)
      if [[ -f "$throttle" && ! -L "$throttle" ]]; then
        mtime=$(stat -c %Y "$throttle" 2>/dev/null || stat -f %m "$throttle" 2>/dev/null || echo 0)
        if (( now - mtime < 300 )); then
          continue   # within 5 min — skip
        fi
      fi
      ```
      (Use the same dual-stat pattern phase-lock.sh uses — Linux GNU stat then BSD stat fallback.)
   c. Call helper:
      ```bash
      set +e
      "$helper" heartbeat "$phase" "claude" >/dev/null 2>&1
      hb_rc=$?
      set -e
      ```
   d. On `hb_rc=0`: refresh throttle. `sb_safe_write "$throttle"; : > "$throttle"` (truncate-create — touch is fine too: `touch "$throttle"` after sb_safe_write).
   e. On any other rc: log `printf '⚠️  heartbeat skipped for phase %s (helper rc=%d)\n' "$phase" "$hb_rc" >&2` and continue to the next phase. Never block.
4. After the loop, exit 0.

Note: heartbeat fires for the CURRENT session's claimed phases (read from manifest), NOT a phase resolved from `tool_input.file_path`. This is intentional per CONTEXT.md `<specifics>`: "For Bash hooks (heartbeat), there is no file_path — heartbeat fires for any active phase in the session manifest." This means: a Bash command on this session refreshes the heartbeat for every phase the session has claimed, regardless of what the command does.

### B. `hooks/phase-lock-release.sh` (HOOK-03)

After the shared prologue:
1. Read stdin JSON (Stop/SubagentStop event), extract `raw_session_id` with the same fallback + sanitization rule. Compute `manifest` path.
2. If `manifest` does not exist, exit 0 (nothing to release).
3. Read each line. For each `phase` (skip blank):

   ```bash
   release_err=$("$helper" release "$phase" "claude" 2>&1 >/dev/null)
   rel_rc=$?

   if (( rel_rc != 0 )); then
     printf '⚠️  phase-lock-release: phase %s rc=%d: %s\n' "$phase" "$rel_rc" "$release_err" >&2
     # Per CONTEXT.md "Manifest cleanup safety": continue clearing other entries even on non-owner.
   fi
   ```
4. After the loop, delete the manifest: `rm -f -- "$manifest"` (only if not a symlink — `[[ -L "$manifest" ]] || rm -f -- "$manifest"`).
5. Exit 0. Stop/SubagentStop must NEVER block (project invariant).

Symlink defense applies only to the throttle file (heartbeat) and manifest read/delete (release). The manifest itself is APPEND-ONLY by claim, READ-ONLY by heartbeat, and READ+UNLINK by release — none of these need `sb_safe_write` after creation, only the claim hook does.
