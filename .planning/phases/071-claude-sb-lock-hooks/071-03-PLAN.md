```bash
# --- Informational lock-owner peek (HOOK-04, non-blocking) -------------------
# Resolves the phase from $PWD; if the project has no lock helper, skip silently.
if command -v jq >/dev/null 2>&1; then
  _lib_dir_peek="$(cd "$(dirname "${BASH_SOURCE[0]}")/lib" && pwd 2>/dev/null)" || _lib_dir_peek=""
  if [[ -n "$_lib_dir_peek" && -f "$_lib_dir_peek/phase-path.sh" ]]; then
    # shellcheck disable=SC1091
    source "$_lib_dir_peek/phase-path.sh"
    _peek_phase=$(_resolve_phase_from_path "$PWD" 2>/dev/null || true)
    _peek_helper="$PWD/.planning/scripts/phase-lock.sh"
    if [[ -n "$_peek_phase" && -x "$_peek_helper" ]]; then
      _peek_owner_json=$("$_peek_helper" peek "$_peek_phase" 2>/dev/null || true)
      if [[ -z "$_peek_owner_json" ]]; then
        printf 'WARN: phase %s has no active lock — proceeding anyway\n' "$_peek_phase" >&2
      else
        _peek_runtime=$(printf '%s' "$_peek_owner_json" | jq -r '.agent_runtime // ""')
        if [[ "$_peek_runtime" != "claude" ]]; then
          _peek_owner_id=$(printf '%s' "$_peek_owner_json" | jq -r '.owner_id // "?"')
          printf 'WARN: phase %s is currently locked by %s — proceeding anyway\n' \
            "$_peek_phase" "$_peek_owner_id" >&2
        fi
      fi
    fi
  fi
fi
# --- end informational peek ---------------------------------------------------
```

Two nuances:
1. Both target hooks have multiple `exit 0` / `exit 1` / `exit N` paths inside `main()`. The peek block must run on EVERY exit path before the script terminates — the cleanest implementation is a function that wraps the existing exit, OR placing the block inside an `EXIT` trap. **Use an `EXIT` trap** (the `trap 'exit 0' ERR` already established in both files is a precedent for trap-based control flow). Set the EXIT trap once near the top of `main()`, immediately AFTER the existing ERR trap, so the peek runs regardless of which path leads to script termination.

2. The peek must NOT alter the exit code. Bash EXIT traps run with `$?` available; the trap body should NOT use `exit` — when an EXIT trap returns naturally, the original `$?` is preserved.

Trap shape:
```bash
_phase_lock_peek_on_exit() {
  local _saved_rc=$?
  # ...peek block from above, with stderr WARN emission only...
  return $_saved_rc   # informational only — preserves original rc
}
trap _phase_lock_peek_on_exit EXIT
```

Note: `completion-audit.sh` and `stop-check.sh` may already register their own
ERR trap (and `stop-check.sh` registers an ERR trap that itself prints JSON and
exits 0 — see line 40 of stop-check.sh). EXIT and ERR are independent traps in
bash; adding an EXIT trap does NOT clobber the ERR trap. Verify by reading the
existing trap lines before inserting.
