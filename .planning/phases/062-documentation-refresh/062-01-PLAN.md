## Silver Bullet vs GSD: Integration Guide

Brief intro (3 sentences): SB and GSD are separate tools with different responsibilities.

## Mental Model: Two Distinct Layers

A short section (4-6 sentences) explaining the architecture:
- SB = enforcement layer + orchestration skills + workflow hooks. Fires on every Claude Code
  tool use. Runs inside the Claude Code session.
- GSD = multi-agent execution engine. Manages planning artifacts (.planning/), spawns subagents
  with fresh 200K contexts, handles context-rot prevention.
- SB calls GSD skills from within workflow steps; GSD does not call SB skills.
- The boundary: SB owns quality gates, compliance state, and workflow sequencing.
  GSD owns execution phases, plan files, wave execution, and git commits.

## Feature Mapping Table

A three-column Markdown table. Columns: Feature | Silver Bullet | GSD
Cover these feature categories (one row per feature):

Enforcement / compliance gates | dev-cycle-check.sh, stop-check.sh, completion-audit.sh | none (no enforcement hooks)
Workflow routing | /silver routes to silver:feature, silver:bugfix, etc. | /gsd:* commands for specific tasks
Quality gates | /silver-quality-gates (9 dimensions), /devops-quality-gates (7 dimensions) | none
Planning artifacts | none (reads .planning/ but does not write it) | ROADMAP.md, PLANS.md, STATE.md, SUMMARYs
Multi-agent execution | dispatches via Agent tool inside silver:* workflows | wave-based parallel execution with fresh contexts
Context rot prevention | none | fresh 200K-token context per agent (GSD's core value)
Blast radius assessment | /silver-blast-radius | none
Release creation | /silver-create-release | /gsd:ship (milestone-level merge and tag)
Session forensics | /silver-forensics | /gsd:forensics (plan-drift, execution anomalies)
Knowledge capture | /silver-rem (lessons/knowledge monthly docs) | none
Issue filing | /silver-add, /silver-remove | none
Retrospective scanning | /silver-scan | none
Skill recording | record-skill.sh (tracks all skills in state file) | tracks GSD-specific skill usage
CI gate | ci-status-check.sh (blocks push/PR/release on red CI) | none
Prompt injection | prompt-reminder.sh (re-injects missing steps before every message) | none
