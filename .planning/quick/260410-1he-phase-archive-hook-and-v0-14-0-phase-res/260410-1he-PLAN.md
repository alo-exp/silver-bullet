# Quick Task: Phase Archive Hook + v0.14.0 Phase Restoration

## Task 1: Create phase-archive PreToolUse hook

**Files:** `hooks/phase-archive.sh` (new)
**Action:** Create a PreToolUse Bash hook that:
- Detects `gsd-tools.cjs phases clear` in the command
- Reads current milestone name from `.planning/PROJECT.md` (grep `Current Milestone:`)
- Creates `.planning/archive/{milestone}/` directory
- Copies all phase directories from `.planning/phases/` to the archive
- Exits 0 to let the original clear command proceed
- Follows existing hook patterns: set -euo pipefail, ERR trap, jq parsing, umask 0077

**Verify:** Hook file exists, is executable, follows SB hook conventions
**Done:** Hook script created

## Task 2: Register hook in hooks.json

**Files:** `hooks/hooks.json` (modify)
**Action:** Add phase-archive.sh as a PreToolUse hook with matcher `Bash` — must fire BEFORE the clear command executes. Place it in the PreToolUse array.

**Verify:** `jq '.hooks[] | select(.event=="PreToolUse") | .hooks[] | select(.command | contains("phase-archive"))' hooks/hooks.json` returns the entry
**Done:** Hook registered

## Task 3: Restore v0.14.0 phases from git history

**Files:** `.planning/archive/v0.14.0/` (new directory tree)
**Action:**
- Find the commit just before phases 12-14 were cleared (the last commit that has those directories)

**Verify:** `ls .planning/archive/v0.14.0/` shows phase 12, 13, 14 directories with their contents
**Done:** v0.14.0 phases restored to archive
