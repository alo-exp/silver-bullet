The Problem

AI Agents Skip Steps — And Lose Context

Even when your CLAUDE.md explicitly says “always plan first,” Claude rationalizes its way past it. And as context fills, quality silently degrades.

Skipped Planning

Claude jumps straight to coding. No brainstorming, no spec, no design review. The path of least resistance is always to just start writing code.

No Quality Gates

Security, scalability, and reliability are afterthoughts — not enforced checkpoints. Modularity and testability are never considered before implementation.

Premature Completion

Work is declared “done” without code review, testing strategy, or verification. Commits happen before the workflow is complete.

Context Rot

As the context window fills, output quality silently degrades. The best workflows in the world mean nothing if the agent is operating on a rotting context.

The root cause: Great workflows exist. Great enforcement tools exist. Great execution engines exist. But they’ve never been combined into a single, coherent, enforced system — until Silver Bullet.

Cost Optimization

Right Model, Right Task, Right Cost

Silver Bullet automatically routes each sub-agent to the optimal model tier. Deep reasoning tasks get Opus. High-throughput tasks get Sonnet. You get 40–60% lower token costs without lifting a finger.

Opus: Deep Reasoning

Planning, architecture decisions, security audits, verification, and quality gates. Tasks where thoroughness and nuance directly impact project quality.

Sonnet: High Throughput

Code execution, documentation, research, testing, and codebase mapping. Tasks where speed and volume matter more than maximum depth.

40–60% Cost Reduction

Running everything on Opus is like using a sports car for grocery runs. Automatic model switching gives each task the right engine — and keeps your token bill from spiraling.

Automatic, not manual. Model assignments are baked into each agent definition. When Silver Bullet spawns a planner, it runs on Opus. When it spawns an executor, it runs on Sonnet. No configuration needed — the routing is built into the system.

Architecture

Six Technical Layers Plus Documentation

Four PostToolUse hooks, a CI status check, session management, and redundant instruction enforcement. Each layer is independently sufficient. Together, they make bypassing the workflow extremely difficult to do by accident.

1

Skill Tracker

A PostToolUse hook fires on every Skill invocation, recording completed steps to a state file. This is the ground truth that every other enforcement layer reads from.

2

Stage Enforcer

A PostToolUse hook fires on every file edit. HARD STOP if quality gates are incomplete. Enforces strict phase ordering: DISCUSS → QUALITY GATES → PLAN → EXECUTE → VERIFY.

3

Compliance Status

Every tool use triggers a progress display: current phase, completed steps, total score, and the next required action. Compliance is always visible — there’s no hiding from it.

4

Completion Audit

A PostToolUse hook intercepts git commit, push, and deploy commands. If the full workflow isn’t complete, the operation is blocked with a list of remaining required steps.

5

CI Status Check

A PreToolUse hook intercepts git push commands and verifies that the most recent CI run for the branch is green before allowing the push. Broken builds cannot be pushed.

6

Session Management

A SessionStart hook tracks the current git branch. When the branch changes between sessions, the full skill-recording state resets — preventing skills from Feature A from satisfying enforcement for Feature B.

7

Redundant Instructions + Anti-Rationalization

The same rules appear in three places: CLAUDE.md, the workflow file, and every hook message. Explicit language names and blocks common excuses: “This is simple enough to skip,” “I’ll do the review mentally.” Named = blocked.

Quality Gates

Quality Dimensions. Enforced Checkpoints.

The full-dev-cycle applies 8 quality dimensions via /quality-gates. The devops-cycle applies 7 IaC-adapted dimensions via /devops-quality-gates (usability excluded) plus a mandatory /blast-radius assessment. Every dimension is a hard stop — not a warning.

Modularity

Small, focused modules. File size limits. Change locality max 3–5 files. Single responsibility enforced.

/quality-gates

Reusability

DRY, composable components. Rule of Three. No duplication across the codebase.

/quality-gates

Scalability

Stateless by default. Indexed queries. Async processing. Resource limits defined upfront.

/quality-gates

Security

OWASP Top 10. Input validation. Secrets management. Defense in depth. No trust boundaries crossed.

/quality-gates

Reliability

Fault tolerance. Retries with backoff. Circuit breakers. Graceful degradation under load.

/quality-gates

Usability

Intuitive APIs. Actionable error messages. Accessibility. Progressive disclosure of complexity.

/quality-gates

Testability

Dependency injection. Pure functions. Test seams. Deterministic behavior. No hidden state.

/quality-gates

Extensibility

Open-closed principle. Plugin architecture. Versioned interfaces. Change without breakage.

/quality-gates

DevOps Cycle: Blast Radius First. Before quality gates, the devops-cycle requires a /blast-radius assessment — mapping change scope, downstream dependencies, failure scenarios, and rollback plans. Changes rated HIGH require explicit approval and a runbook. CRITICAL changes are a hard stop requiring CAB review.

Installation

Get Started in Two Minutes

Install the four core plugins, add Silver Bullet, and initialize. For DevOps projects, optionally add leading IaC plugins for contextual enrichment.

# 1. Install the four core workflow plugins
npx get-shit-done-cc@1.30.0
/plugin install obra/superpowers
/plugin install anthropics/knowledge-work-plugins/tree/main/engineering
/plugin install anthropics/knowledge-work-plugins/tree/main/design
 
# 2. Install Silver Bullet
/plugin install alo-exp/silver-bullet
 
# 3. (Optional) DevOps plugins — for devops-cycle workflow
/plugin marketplace add hashicorp/agent-skills
/plugin marketplace add awslabs/agent-plugins
/plugin marketplace add pulumi/agent-skills
/plugin marketplace add ahmedasmar/devops-claude-skills
/plugin marketplace add wshobson/agents
 
# 4. Initialize your project
/using-silver-bullet
# Asks: application or DevOps/infrastructure project?

Enforcement starts immediately.
After running /using-silver-bullet, hooks are active and the workflow is live. For app projects, the 20-step full-dev-cycle activates. For DevOps projects, the 24-step devops-cycle activates with blast radius assessment, IaC-adapted quality gates, and automatic detection of installed DevOps plugins.
