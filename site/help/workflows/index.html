Orchestration Workflows

Silver Bullet provides seven pre-designed orchestration workflows. /silver routes to the right one automatically based on your intent — or invoke any workflow directly.

The smart entry point: type /silver [your intent in plain English] and Silver Bullet classifies your request and dispatches to the right workflow automatically. No need to remember which workflow to use.
Documentation

Seven Workflows

Each workflow chains GSD (execution backbone), Superpowers (craft discipline), MultAI (multi-AI intelligence), and SB quality gates in a defined sequence. None implement code directly — they orchestrate only.

Full workflow

/silver:feature

Full feature development cycle — from codebase intel through quality gates, TDD, code review, and ship.

  • Complexity triage (trivial/simple/fuzzy/complex)
  • product-brainstorming + engineering brainstorm
  • 9-dimension quality gates (pre-plan + pre-ship)
  • TDD gate for implementation plans
  • Full code review sequence

Triage-first

/silver:bugfix

Triage-first bug investigation — classifies failure type into Path 1A, 1B, or 1C before any investigation begins.

  • Path 1A: Known symptom → systematic-debugging
  • Path 1B: Unknown cause → silver:forensics
  • Path 1C: GSD workflow failure → gsd-forensics
  • TDD regression test before writing any fix
Read workflow →
UI/UX

/silver:ui

UI/frontend workflow with design contract (gsd-ui-phase), component TDD, 6-pillar visual audit, and accessibility enforcement.

  • gsd-ui-phase creates UI-SPEC.md design contract
  • TDD for testable component units
  • gsd-ui-review: 6-pillar visual audit
  • Accessibility (WCAG 2.1 AA) enforcement
Read workflow →
DevOps

/silver:devops

Infrastructure, CI/CD, and IaC workflow — blast-radius assessment mandatory, 7 IaC quality dimensions, TDD explicitly skipped.

  • blast-radius: LOW / MEDIUM / HIGH / CRITICAL
  • devops-skill-router selects right IaC tooling
  • 7 IaC quality dimensions (no usability/extensibility)
  • TDD explicitly not applicable for infra

Research

/silver:research

Technology decisions, architecture spikes, and competitive analysis — MultAI research paths with handoff to implementation.

  • Path 2a: market/landscape research
  • Path 2b: tech selection (7-AI comparison matrix)
  • Path 2c: competitive intelligence
  • Handoff to silver:feature or silver:devops
Read workflow →
Milestone

/silver:release

Milestone-level publish — versioned releases, changelogs, docs, GitHub Releases, and milestone archival. Not the same as gsd:ship.

  • Pre-release quality gates (9 dimensions)
  • Cross-phase UAT + milestone audit
  • Gap-closure loop (max 2 iterations)
  • silver:create-release → gsd-ship → complete-milestone
Read workflow →
Fast path

/silver:fast

Bypass workflow for truly trivial changes (≤3 files): typos, config values, renames. No planning, gates, or review cycle.

  • Complexity triage gate before gsd-fast
  • Scope expansion STOP condition (3-file limit)
  • §10 preferences NOT applied (by design)
  • Auto-escalation if scope grows
Read workflow →

Just use /silver — you don't need to choose a workflow manually. Silver Bullet classifies your intent (trivial/simple/fuzzy/complex) and dispatches to the right workflow automatically. The seven workflows are named here so you understand what's happening, not so you have to memorize them.
