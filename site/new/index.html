The Problem

AI Agents Skip Steps — And Lose Context

Even when your CLAUDE.md explicitly says “always plan first,” Claude rationalizes its way past it. And as context fills, quality silently degrades.

Skipped Planning

Claude jumps straight to coding. No brainstorming, no spec, no design review. The path of least resistance is always to just start writing code.

No Quality Gates

Security, scalability, and reliability are afterthoughts — not enforced checkpoints. Modularity and testability are never considered before implementation.

Premature Completion

Work is declared “done” without code review, testing strategy, or verification. Commits happen before the workflow is complete.

Context Rot

As the context window fills, output quality silently degrades. The best workflows in the world mean nothing if the agent is operating on a rotting context.

The root cause: Great workflows exist. Great enforcement tools exist. Great execution engines exist. But they’ve never been combined into a single, coherent, enforced system — until Silver Bullet.

Artifact Review System

Every Artifact Reviewed. Every Time.

Eight domain-specific reviewer skills enforce quality across the full planning artifact chain. Two consecutive clean passes required before approval. No exceptions.

8 Artifact Reviewers

SPEC, DESIGN, REQUIREMENTS, ROADMAP, CONTEXT, RESEARCH, INGESTION_MANIFEST, and UAT — each reviewer runs domain-specific QC checks tailored to its artifact type.

2-Consecutive-Clean-Pass Enforcement

Every artifact must pass two consecutive clean review rounds before it is approved. State is tracked per artifact using SHA256-keyed JSON — no gaming the system.

Configurable Review Depth

Configure deep, standard, or quick review depth per artifact type in .planning/config.json. Standard is the default for backward compatibility.

Cross-Artifact Consistency

Validates SPEC-to-REQUIREMENTS-to-ROADMAP-to-DESIGN alignment in a single pass. Wired into milestone completion and the release workflow — misaligned artifacts block shipping.

Architecture

Ten Enforcement Layers

Skill tracker, stage enforcer, compliance status, completion audit, CI gate, session management, Stop hook, UserPromptSubmit reminder, forbidden-skill gate, and redundant instruction enforcement. Each layer is independently sufficient.

1

Skill Tracker

A PostToolUse hook fires on every Skill invocation, recording completed steps to a state file. This is the ground truth that every other enforcement layer reads from.

2

Stage Enforcer

A PostToolUse hook fires on every file edit. HARD STOP if quality gates are incomplete. Enforces strict phase ordering: DISCUSS → QUALITY GATES → PLAN → EXECUTE → VERIFY.

3

Compliance Status

Every tool use triggers a progress display: current phase, completed steps, total score, and the next required action. Compliance is always visible.

4

Completion Audit

A PostToolUse hook intercepts git commit, push, and deploy commands. If the full workflow isn’t complete, the operation is blocked with a list of remaining required steps.

5

CI Status Check

A PreToolUse hook intercepts git push commands and verifies that the most recent CI run for the branch is green before allowing the push.

6

Session Management

A SessionStart hook tracks the current git branch. When the branch changes between sessions, the full skill-recording state resets — preventing cross-feature contamination.

7

Stop Hook

A Stop/SubagentStop hook fires when Claude tries to declare a task complete. If any required skills are missing from the state file, the declaration is blocked.

8

Prompt Reminder

A UserPromptSubmit hook re-injects the missing skills list and core enforcement rules before every user message — ensuring each new prompt starts with full context.

9

Forbidden Skill Gate

A PreToolUse hook intercepts every Skill invocation. Deprecated or project-forbidden skills are blocked before they execute.

10

Redundant Instructions + Anti-Rationalization

The same rules appear in three places: CLAUDE.md, the workflow file, and every hook message. Named excuses are explicitly blocked.

Installation

Get Started in Two Minutes

Install the four core plugins, add Silver Bullet, and initialize. For DevOps projects, optionally add leading IaC plugins for contextual enrichment.

# 1. Install the four core workflow plugins
npx get-shit-done-cc@1.30.0
/plugin install obra/superpowers
/plugin install anthropics/knowledge-work-plugins/tree/main/engineering
/plugin install anthropics/knowledge-work-plugins/tree/main/design
 
# 2. Install Silver Bullet
/plugin install alo-exp/silver-bullet
 
# 3. Initialize your project
/silver:init
 
# 4. Use /silver as your single entry point
/silver quality review my architecture

Enforcement starts immediately.
After running /silver:init, hooks are active and the workflow is live. Use /silver <anything> as your single entry point — it routes to the right skill automatically.
