Layer 4

Language choice — orders of magnitude in energy cost

Pereira et al., ACM SIGPLAN 2017 · Revisited 2024
75×
Python vs. C — benchmark energy gap
On compute-heavy algorithmic benchmarks (CLBG). Peer-reviewed: Pereira et al., ACM SLE 2017, DOI: 10.1145/3136014.3136031. Validated in 2021 followup.
Pereira et al., ACM SIGPLAN 2017 · Science of Computer Programming 2021
~4×
Real-world Python vs. C gap
In production code with I/O, syscalls, and library calls (not pure compute). Still significant at scale — green-coding.io empirical study, 2024
Green-Coding.io, 2024 (revisit of Pereira)
76.6%
All websites running PHP
An interpreted language with significant runtime overhead — powering the majority of the web's backend in 2025
JetBrains State of PHP, 2025
2.4–119×
Performance improvement compiling Python vs. standard CPython runtime
Showing how much efficiency is left on the table by the default language runtime — yet standard deployments use CPython universally
arXiv:2505.02346, May 2025
⚠ Caveat: the 75× figure is benchmark-specific (compute-intensive algorithms). Real-world applications with I/O-heavy workloads show a ~4× gap. Both figures come from the same peer-reviewed source; context matters when citing.
